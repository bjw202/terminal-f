# terminal-f 명령 팔레트 완전 안내 (쉬운 설명판)

> `Ctrl+Shift+P`를 누르면 뜨는 **명령 팔레트**의 모든 메뉴를, 각각
> **① 하는 일 · ② 쓰는 법 · ③ 어떻게 동작하나(원리)** 로 나눠 설명합니다.
> 개발 지식이 없어도 읽을 수 있게 썼습니다.
>
> 짧은 한 줄 요약만 필요하면 [기능 가이드 §6 메뉴 사전](GUIDE-features-easy.md#6-명령-팔레트-메뉴-사전)을 보세요.
> 이 문서는 그보다 **자세한 설명서**입니다.

---

## 명령 팔레트가 뭔가요?

터미널의 **만능 검색창**입니다. 마우스로 메뉴를 뒤질 필요 없이, 하고 싶은
일을 몇 글자만 치면 관련 명령이 뜨고 Enter로 실행됩니다.

- **여는 법**: `Ctrl+Shift+P`
- **찾는 법**: 아무 단어나 입력 (예: `split`, `테마`는 안 되고 영어 `theme`,
  `font`, `inject`…). 명령 이름의 일부만 쳐도 걸러집니다.
- **실행**: 위/아래 방향키로 고르고 `Enter`. 닫기는 `Esc`.
- 어떤 명령은 **지금 보고 있는 칸**이나 **지금 책상**을 대상으로 동작합니다.
  그래서 팔레트에 뜨는 목록은 상황에 따라 조금씩 달라집니다(예: 다른 책상이
  있을 때만 "그 책상으로 이동" 메뉴가 보임).

아래 명령들은 주제별로 묶었습니다. **매일 쓰는 것(화면·책상·보기·복사)** 이
위쪽에, **자동화·외부연결 같은 고급 기능**이 아래쪽에 있습니다.

---

## 1. 화면(칸) 다루기 — 매일 쓰는 것

터미널 창을 여러 "칸(pane)"으로 쪼개 여러 프로그램을 나란히 봅니다.

### Pane: Split left/right  ·  단축키 `Ctrl+Shift+D`
- **하는 일**: 지금 보고 있는 칸을 **좌우 둘**로 나눕니다.
- **쓰는 법**: 나누고 싶은 칸을 클릭해 두고 실행(또는 단축키). 새 칸이
  오른쪽에 생기고, 그 안에서 새 명령창(PowerShell)이 켜집니다.
- **원리**: 화면을 반으로 접듯 나누고, 각 칸은 **완전히 독립된 명령창**을
  가집니다. 한 칸에서 하는 일은 다른 칸에 영향을 주지 않습니다. 새 칸이 어느
  폴더에서 열리는지는 §5의 "live directory tracking" 설정에 따라 달라집니다.

### Pane: Split top/bottom  ·  단축키 `Ctrl+Shift+-`
- **하는 일**: 지금 칸을 **위아래 둘**로 나눕니다. (좌우가 아니라 상하)
- **쓰는 법**: 위와 같되 결과가 위/아래로 쌓입니다.
- **원리**: 좌우 나누기와 똑같고 방향만 다릅니다. 나누기를 반복하면 바둑판
  처럼 여러 칸을 만들 수 있고, 칸 사이 경계선을 마우스로 끌어 크기를 조절할
  수 있습니다.

### Pane: Close focused  ·  단축키 `Ctrl+Shift+W`
- **하는 일**: 지금 칸 하나를 닫습니다.
- **쓰는 법**: 닫을 칸을 클릭해 두고 실행. 마지막 남은 칸은 실수 방지를 위해
  닫히지 않습니다(책상 자체를 비우지 않도록).
- **원리**: 그 칸의 명령창을 종료하고, 남은 칸들이 빈자리를 채워 배치가 자동
  으로 다시 정렬됩니다.

### Pane: Toggle zoom  ·  단축키 `Ctrl+Shift+Z`
- **하는 일**: 지금 칸을 **책상 전체로 확대** ↔ 원래 배치로 되돌리기.
- **쓰는 법**: 한 칸에 집중하고 싶을 때 실행하면 그 칸만 꽉 차게 보입니다.
  다시 실행하면 원래 여러 칸 배치로 돌아옵니다.
- **원리**: 다른 칸을 닫는 게 아니라 **잠깐 숨기는** 것입니다. 숨은 칸의
  프로그램은 계속 돌아가고, 확대를 풀면 그대로 다시 나타납니다.

## 2. 책상(워크스페이스) 다루기

**워크스페이스 = 책상**. 프로젝트마다 하나씩 두고, 각 책상에 칸을 여러 개
둡니다. 책상을 옮겨도 이전 책상의 작업은 안 멈춥니다(keep-alive).

### Workspace: New
- **하는 일**: 빈 새 책상을 만듭니다.
- **쓰는 법**: 실행하면 새 책상이 생기고 그리로 이동합니다. 사이드바에 새
  항목으로 나타납니다.
- **원리**: 각 책상은 자기만의 칸 배치와 명령창들을 가진 독립 공간입니다.
  책상을 껐다 켜도 배치는 저장되어 복원됩니다(단, 돌아가던 프로그램 자체는
  종료됨).

### Workspace: Switch to "이름"  ·  단축키 `Ctrl+1` ~ `Ctrl+8`
- **하는 일**: 그 이름의 책상으로 이동합니다.
- **쓰는 법**: 이 메뉴는 **다른 책상이 있을 때만** 목록에 뜹니다. 1~8번
  책상은 `Ctrl+숫자`로 즉시 이동할 수도 있습니다.
- **원리**: 화면에 보이는 책상만 바뀔 뿐, **안 보이는 책상의 작업은 계속
  진행**됩니다(keep-alive). 돌아오면 그동안 밀린 출력까지 다시 보입니다.
  너무 많이 쌓이면 가장 오래된 부분은 잘리고 "잘렸음" 표시가 뜹니다.

## 3. 보기(화면 모양) 바꾸기

### View: Toggle sidebar  ·  단축키 `Ctrl+Shift+B`
- **하는 일**: 왼쪽 책상 목록 사이드바를 **접기/펴기**.
- **쓰는 법**: 화면을 넓게 쓰고 싶으면 접어서 좁은 아이콘 줄만 남깁니다.
- **원리**: 접어도 정보는 그대로 있고 표시만 좁아집니다. 안 보는 책상에서
  새 출력이 나오면 사이드바 이름 옆에 작은 점이, 프로그램이 죽으면 빨간 점이
  켜져서 접어놔도 상태를 알 수 있습니다.

### View: Increase font size  /  View: Decrease font size
- **하는 일**: 터미널 글자를 한 단계 크게 / 작게.
- **쓰는 법**: 눈이 편한 크기가 될 때까지 반복 실행. 설정은 저장됩니다.
- **원리**: 글자 크기에 맞춰 각 칸이 담을 수 있는 글자 수(가로·세로 칸)가
  자동으로 다시 계산되어, 안에서 돌아가는 프로그램도 새 크기에 맞춰집니다.

### Theme: … (색 테마)
팔레트에서 `theme`을 치면 아래 테마들이 뜹니다. 고르면 즉시 적용됩니다.

| 메뉴 | 설명 |
|---|---|
| **Theme: Catppuccin Mocha (dark)** | 부드러운 어두운 테마(기본) |
| **Theme: Catppuccin Latte (light)** | 부드러운 밝은 테마 |
| **Theme: One Dark** | 코드 편집기 느낌의 어두운 테마 |
| **Theme: Solarized Light** | 눈에 편한 밝은 테마 |
| **Theme: Campbell PowerShell (navy)** | Windows Terminal 기본 PowerShell과 똑같은 남색 배경 |

- **원리**: 배경색·글자색·16색 팔레트를 통째로 바꿉니다. 고른 테마는 저장
  되어 다음에 켜도 유지됩니다.

## 4. 복사·붙여넣기 편의

### Copy: Enable / Disable copy-on-select
- **하는 일**: **글자를 마우스로 선택하는 순간 자동으로 복사**되게 켜기/끄기.
  (기본은 꺼짐)
- **쓰는 법**: 켜두면 드래그만 해도 복사됩니다(`Ctrl+C`를 안 눌러도 됨).
  메뉴 이름은 현재 상태에 따라 "Enable"↔"Disable"로 바뀝니다.
- **원리**: 리눅스 터미널의 익숙한 습관입니다. 실수로 원치 않게 복사되는 게
  싫으면 꺼두세요. 켜져 있어도 붙여넣기는 평소대로 `Ctrl+V`/우클릭입니다.

> 참고: 복사·붙여넣기 전반(선택 후 `Ctrl+C`, `Ctrl+Shift+C`, 우클릭, TUI에서
> `Shift`+드래그, OSC 52)은 [기능 가이드 §5](GUIDE-features-easy.md#5-복사붙여넣기파일-놓기)에 자세히 있습니다.

## 5. PowerShell 셸 통합 (한 번만 설정하는 편의)

이 두 개는 **PowerShell 프로필**(셸이 켜질 때 자동 실행되는 개인 설정 파일)에
작은 설정을 한 줄 추가하는 명령입니다. 파일을 건드리므로 **추가할 내용을 먼저
보여주고 동의를 받은 뒤**에만 설치합니다. 설치 후에는 **새 칸을 열어야**
적용됩니다(프로필은 셸이 처음 켜질 때만 읽히기 때문).

### Shell: Enable multiline in PowerShell (Ctrl+Enter)
- **하는 일**: 일반 PowerShell 프롬프트에서도 `Ctrl+Enter`/`Shift+Enter`로
  **줄바꿈**이 들어가게 합니다(엔터로 바로 실행되지 않고 다음 줄로).
- **쓰는 법**: 실행 → 안내 창에서 추가될 내용을 확인하고 [Install] →
  **새 PowerShell 칸**을 열면 적용. 되돌리려면 프로필에서 해당 블록 삭제.
- **원리**: 터미널은 `Ctrl+Enter` 같은 조합을 그대로 전달할 방법이 없어서,
  terminal-f는 이를 `Alt+Enter`라는 신호로 바꿔 보냅니다. 이 설정은
  PowerShell에게 "`Alt+Enter`가 오면 줄바꿈으로 처리해"라고 알려주는 것뿐
  입니다. (Claude Code 같은 도구 안에서는 이 설정 없이도 이미 동작합니다.)

### Shell: Enable live directory tracking in PowerShell (split follows cwd)
- **하는 일**: 칸을 나눌 때(split) 새 칸이 **지금 내가 `cd`로 이동해 있는
  폴더**에서 열리게 합니다(처음 열렸던 폴더가 아니라).
- **쓰는 법**: 실행 → 안내 창 확인 후 [Install] → **새 PowerShell 칸**을 열고
  → `cd`로 이동 → split 하면 그 폴더에서 열립니다.
- **이미 설치돼 있는데 예전 버전이면**: 이 명령을 다시 실행하면 "이미
  설치됨" 대신 **[Update]** 창이 떠서 최신 버전으로 갱신됩니다.
- **원리**: Windows에서 PowerShell은 `cd`를 해도 프로그램의 실제 위치가 안
  바뀌어서, terminal-f가 밖에서 현재 폴더를 알아낼 방법이 없습니다. 그래서
  이 설정을 켜면 PowerShell이 **프롬프트를 그릴 때마다 "나 지금 이 폴더야"
  라는 눈에 안 보이는 신호**를 함께 내보냅니다(Windows Terminal·VS Code와
  같은 방식). terminal-f가 그 신호를 읽어 기억해 두었다가 새 칸을 그 폴더에서
  엽니다. 이 신호는 화면엔 안 보이게 걸러집니다. **PowerShell 전용**이며
  cmd는 아직 지원하지 않습니다.

## 6. 매크로(레시피/템플릿) — 책상 세팅 자동화

매일 아침 "책상 만들고 → 칸 나누고 → 폴더 이동하고 → AI 도구 켜고"를 손으로
하던 걸 **레시피 하나로** 대신합니다. 자세한 배경은
[기능 가이드 §2](GUIDE-features-easy.md#2-새-기능--매크로-버튼-하나로-책상-자동-세팅)에 있습니다.

### Template: Save current layout as template
- **하는 일**: **지금 책상의 칸 배치**를 레시피로 저장합니다.
- **쓰는 법**: 원하는 배치를 만든 뒤 실행 → 이름 입력. 이후 팔레트에
  `Template: Apply "그 이름"` 으로 나타납니다.
- **원리**: 칸을 어떻게 나눴는지(가로/세로/비율)와 각 칸의 정보를 그림 설계도
  처럼 저장합니다. 나중에 그 설계도대로 새 책상을 조립합니다.

### Template: Apply "이름"
- **하는 일**: 저장된 레시피를 실행해 **완성된 새 책상**을 엽니다(칸 나누기 +
  폴더 이동 + 프로그램 실행까지).
- **쓰는 법**: 실행 → 레시피에 물어볼 항목(예: 폴더)이 있으면 답하면 끝.
- **원리**: 저장해 둔 설계도를 읽어 칸을 자동으로 나누고, 각 칸에서 정해진
  명령을 실행합니다. 레시피 안의 `${repo}` 같은 빈칸은 실행할 때 물어봐
  채웁니다.

### Template: Apply repo profile (.terminal-f/profile.json)
- **하는 일**: **프로젝트 폴더 안에 들어 있는 공유 레시피**를 실행합니다.
  (팀원과 함께 쓰는, 프로젝트에 딸려오는 레시피)
- **쓰는 법**: 실행 → 그 프로젝트 폴더를 지정. 자동 실행 명령이 들어 있으면
  **"이 프로젝트를 신뢰하시겠습니까?" 창**이 먼저 뜹니다.
- **원리(중요-안전)**: 인터넷에서 받은 낯선 프로젝트의 레시피에 나쁜 명령이
  숨어 있을 수 있으므로, **처음 여는 프로젝트는 신뢰 확인 전에는 어떤 명령도
  자동 실행하지 않습니다.** 낯선 USB를 꽂기 전에 확인하는 것과 같습니다. 한
  번 신뢰하면 다음부터는 묻지 않습니다.

## 7. 자동 입력(프롬프트 주입) — AI 칸에 대신 타이핑

자동화가 특정 칸에 텍스트를 **대신 입력**해 주는 기능의 스위치·수동버튼·
안전장치입니다. **아무것도 안 켜면 완전히 꺼져 있는 것과 같습니다.** 배경은
[기능 가이드 §3](GUIDE-features-easy.md#3-새-기능--ai-비서들끼리-협업시키기-자동-프롬프트-전달)에 있습니다.

### Injection: Allow / disallow on focused pane
- **하는 일**: 지금 칸의 **"자동 입력 허용" 스위치**를 켜기/끄기.
  **기본은 꺼짐.**
- **쓰는 법**: 자동화가 입력해도 되는 칸(예: Codex가 떠 있는 칸)에서 실행해
  켭니다. 켜진 칸은 이름표에 ⚡ 표시가 붙습니다.
- **원리**: 이 스위치가 꺼진 칸에는 **어떤 자동화도(규칙이든 외부
  프로그램이든) 절대 입력하지 못합니다.** 안전의 1차 관문입니다.

### Pane: Edit labels (focused)
- **하는 일**: 지금 칸에 **이름표(라벨)**를 붙입니다(예: `codex`,
  `reviewer`).
- **쓰는 법**: 실행 → 쉼표로 구분해 여러 개 입력 가능(`codex, reviewer`).
- **원리**: 자동화 규칙이 대상 칸을 **이름표로** 찾습니다("codex 라벨이 붙은
  칸에 보내라"). 칸 위치가 바뀌어도 이름표로 정확히 찾아갑니다.

### Injection: Send prompt to pane N [라벨]
- **하는 일**: 사람이 **직접** 특정 칸에 프롬프트 한 개를 보내는 수동 버튼.
- **쓰는 법**: 이 메뉴는 **"자동 입력 허용"이 켜진 칸에 대해서만** 목록에
  뜹니다. 실행 → 보낼 문구 입력 → 그 칸에 입력됩니다.
- **원리**: 규칙·감시원 없이도 주입 기능을 손으로 써 볼 수 있는 통로입니다.
  허용 스위치가 꺼진 칸은 애초에 목록에 나오지 않습니다.

### Injection: Pause all (kill switch)  /  Resume
- **하는 일**: **비상 정지.** 모든 자동 입력을 즉시 일시정지합니다. 다시
  실행하면 재개(메뉴 이름이 상태에 따라 바뀜).
- **쓰는 법**: 자동화가 이상하게 동작하는 것 같으면 바로 실행. 화면 하단
  버튼으로도 됩니다.
- **원리**: 모든 경로(규칙·외부 프로그램)의 자동 입력을 한 번에 막습니다.
  **내가 직접 키보드로 치는 것은 전혀 영향받지 않습니다** — 순수하게 자동
  입력만 멈춥니다.

### Injection: Show audit log
- **하는 일**: 자동 입력의 **전체 기록장**을 엽니다.
- **쓰는 법**: 실행하면 "언제 / 누가(수동·규칙·외부) / 어느 칸에 / 뭐라고"
  입력했는지 목록이 나옵니다.
- **원리**: 모든 주입은 실행 전에 기록됩니다. 자동화가 무슨 일을 했는지
  나중에 100% 되짚어 볼 수 있어, 안전 기능의 마지막 확인 수단입니다.

## 8. 자동화 규칙(감시원) — "무엇이 일어나면 → 무엇을 한다"

"이런 일이 생기면 저 칸에 이렇게 입력하라"는 약속을 등록합니다. **규칙이
발동해도 기본은 [승인]/[무시] 확인 창이 먼저** 뜨고, "완전 자동"은 규칙마다
직접 켜야 합니다.

### Automation: Add git-review rule (watch folder → inject to label)
- **하는 일**: **"이 폴더의 코드가 바뀌면, 저 라벨 칸에 리뷰 요청을 보내라"**
  는 규칙을 만듭니다.
- **쓰는 법**: 실행 → ① 감시할 git 프로젝트 폴더 지정(기본은 지금 칸의 폴더)
  → ② 어느 라벨 칸에 보낼지 입력(예: `codex`). "auto mode" 체크를 안 하면
  발동 시 **확인 창**이, 체크하면 **묻지 않고 바로** 보냅니다.
- **원리**: 감시원이 그 폴더의 코드 변경(작업 트리 변경)을 지켜보다가, 바뀌면
  "작업 트리가 바뀌었어요: (요약)\n리뷰해 주세요" 같은 문구를 그 라벨 칸에
  입력합니다. 폭주를 막으려고 **한 번 보낸 뒤 잠깐 쉬고(약 5초), 분당 횟수도
  제한**하며, 대상 칸이 **한창 일하는 중이면 조용해질 때까지 기다립니다.**
  대상 라벨 칸은 반드시 "자동 입력 허용"이 켜져 있어야 합니다.

### Automation: Add timer rule (inject every N minutes → label)
- **하는 일**: **"N분마다 저 라벨 칸에 이 문구를 보내라"**는 규칙을 만듭니다.
- **쓰는 법**: 실행 → ① 몇 분 간격인지 → ② 어느 라벨 칸에 → ③ 보낼 문구.
  정기 상태 점검("진행 상황 알려줘") 등에 씁니다.
- **원리**: 정해진 시간마다 문구를 그 칸에 입력합니다. git 규칙과 같은
  안전장치(쿨다운·대상 칸 유휴 대기·허용 스위치 필요·확인/자동 선택)가 똑같이
  적용됩니다.

### Automation: List rules
- **하는 일**: 등록된 규칙 목록을 봅니다.
- **원리**: 지금 어떤 감시원이 무엇을 지켜보고 있는지 한눈에 확인하는
  화면입니다.

### Automation: Run rule now — 이름
- **하는 일**: 그 규칙을 기다리지 않고 **지금 바로** 한 번 발동시킵니다.
- **쓰는 법**: 규칙을 새로 만든 뒤 잘 동작하는지 **시험**할 때 유용합니다.
- **원리**: 조건 충족을 기다리지 않고 즉시 실행하되, 실제 발동과 똑같이 확인
  창·안전장치를 거칩니다.

### Automation: Enable / Disable rule — 이름
- **하는 일**: 규칙을 **지우지 않고** 잠시 켜기/끄기.
- **원리**: 껐다가 나중에 다시 켤 수 있어, 규칙을 매번 다시 만들 필요가
  없습니다.

### Automation: Remove rule — 이름
- **하는 일**: 규칙을 완전히 삭제합니다.

## 9. 외부 연결(고급) — 개발자용

프로그래밍을 아는 사용자가 직접 만든 감시원 프로그램을 terminal-f에 붙이는
통로입니다. 일반 사용자는 몰라도 됩니다.

### Observe: Allow / disallow output observation on focused pane
- **하는 일**: 외부 프로그램이 이 칸의 **화면 출력을 읽어가도 되는지** 스위치.
  **기본은 꺼짐.**
- **쓰는 법**: 실행해 켜면 이름표에 👁 표시가 붙습니다.
- **원리(중요-보안)**: 터미널 화면에는 비밀번호·토큰 같은 민감한 내용이 지나갈
  수 있습니다. 그래서 **명시적으로 켠 칸의 출력만** 외부가 읽을 수 있고,
  기본은 꺼져 있어 아무것도 새어 나가지 않습니다.

### Control API: Show connection info (named pipe)
- **하는 일**: 외부 프로그램이 terminal-f에 접속할 **통로 주소와 안내**를
  보여줍니다.
- **쓰는 법**: 직접 브로커/스크립트를 만드는 사람이 접속 정보를 확인할 때
  씁니다.
- **원리(중요-보안)**: 접속에는 비밀 토큰이 필요한데, 이 창에는 **주소만 보여
  주고 토큰 자체는 화면에 노출하지 않습니다**(어깨너머로 유출되는 것 방지).
  외부에서 들어온 입력도 §7·§8의 **모든 안전장치를 똑같이 통과**해야 하며,
  우회할 수 없습니다.

## 한눈에 보는 안전 원칙 (자동화·외부연결)

1. **허락한 칸에만**: "자동 입력 허용"이 켜진 칸에만 입력됩니다(기본 꺼짐).
2. **끼어들기 금지**: 대상 칸이 일하는 중이면 조용해질 때까지 기다립니다.
3. **승인이 기본**: 규칙이 발동해도 확인 창이 먼저 뜹니다(완전 자동은 옵트인).
4. **전부 기록**: 모든 주입은 감사 로그에 남습니다.
5. **폭주 방지**: 한 번 발동한 규칙은 잠시 쉬고, 분당 횟수도 제한됩니다.
6. **비상 정지**: 버튼 하나로 모든 자동화를 즉시 멈춥니다(내 타이핑은 무관).
7. **출력 관찰도 옵트인**: 화면 내용을 외부가 읽는 것도 켠 칸만(기본 꺼짐).
8. **낯선 프로젝트는 신뢰 확인 후에만** 자동 실행합니다.
