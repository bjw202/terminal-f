# terminal-f 기능 가이드 (쉬운 설명판)

> 개발 지식이 없어도 읽을 수 있도록 쓴 문서입니다. 각 기능이
> "무엇이고, 왜 필요하고, 어떻게 쓰는지"를 설명합니다.
> (여기 소개된 기능은 모두 구현되어 있습니다. 기술 상세가 궁금하면
> [PLAN-M1-M2-roadmap.md](PLAN-M1-M2-roadmap.md)를 보세요.)
>
> **모든 메뉴의 빠른 사전은 [§6. 명령 팔레트 메뉴 사전](#6-명령-팔레트-메뉴-사전)에 있습니다.**

---

## 0. 먼저, 지금의 terminal-f를 한 문장으로

**"작업용 책상(워크스페이스)을 여러 개 두고, 각 책상 위에 명령창(터미널)을
여러 칸으로 쪼개 놓을 수 있는 프로그램"** 입니다.

| 용어 | 쉬운 비유 |
|---|---|
| 워크스페이스 | 책상. 프로젝트마다 하나씩 두는 것을 권장 |
| 페인(pane) | 책상 위 화면 분할 칸. 한 칸 = 명령창 하나 |
| 세션 | 각 칸 안에서 실제로 돌아가는 프로그램(셸) |
| keep-alive | 다른 책상으로 자리를 옮겨도, 원래 책상 위의 일은 **멈추지 않고 계속 돌아감** |

지금도 되는 것: 책상 만들기/이름 바꾸기/버리기, 칸 나누기(가로/세로)와
닫기, 책상 사이 이동, 프로그램을 껐다 켜도 책상 배치가 그대로 복원.

## 1. 새 기능 ①: 왼쪽 사이드바와 테마

### 무엇이 바뀌나

- 지금은 책상 목록이 **위쪽 탭**으로 나열됩니다. 앞으로는 **왼쪽 세로
  사이드바**로 바뀝니다. 책상이 많아져도 한눈에 보입니다.
- 사이드바는 **접을 수 있습니다.** 접으면 좁은 아이콘 줄만 남아서 화면을
  거의 차지하지 않고, 펼치면 이름과 상태가 다 보입니다. (단축키 `Ctrl+B`)
- 색 테마를 고를 수 있게 됩니다(어두운 테마, 밝은 테마 등). 글꼴과 크기도
  설정에서 바꿀 수 있습니다.

### 눈여겨볼 편의 기능

- **활동 표시등**: 지금 안 보고 있는 책상에서 뭔가 새 출력이 나오면,
  사이드바의 그 책상 이름 옆에 **작은 점**이 켜집니다. "저 책상에서 뭔가
  일어났구나"를 바로 알 수 있습니다.
- **빨간 표시**: 어떤 칸의 프로그램이 종료되면(죽으면) 빨간 점으로 알려줍니다.
- `Ctrl+1`~`Ctrl+8`: 1번~8번 책상으로 즉시 이동.
- **명령 팔레트**(`Ctrl+Shift+P`): 모든 기능을 검색해서 실행하는 만능
  검색창. "화면 나누기", "테마 바꾸기", 아래에서 설명할 "매크로 실행"까지
  전부 여기서 타이핑 몇 글자로 실행할 수 있습니다.

## 2. 새 기능 ②: 매크로 (버튼 하나로 책상 자동 세팅)

### 어떤 불편을 없애나

매일 아침 같은 프로젝트를 열 때마다:
"책상 만들고 → 칸을 3개로 나누고 → 각 칸에서 폴더 이동하고 → 첫 칸엔
Claude Code 켜고, 둘째 칸엔 Codex 켜고, 셋째 칸은 그냥 명령창으로..."
이걸 매번 손으로 합니다.

### 무엇이 되나

이 세팅 과정을 **"레시피"(프로필)로 저장**해 둡니다. 다음부터는:

1. 명령 팔레트에서 레시피 이름 실행 (예: "ai-pair-dev")
2. 폴더를 물어보면 선택
3. 끝. 칸 나누기 + 폴더 이동 + 프로그램 실행까지 자동으로 완성된 새 책상이 뜸

지금 쓰고 있는 책상 배치를 **"이 배치를 레시피로 저장"** 한 번으로 만들 수도
있습니다.

### 레시피는 두 군데 저장됩니다

- **내 컴퓨터 전용**: 나만 쓰는 개인 레시피.
- **프로젝트 폴더 안**: 팀원과 공유하는 레시피. 프로젝트를 받으면 레시피도
  따라옵니다.

### "신뢰 확인" 창이 뜨는 이유 (중요)

프로젝트 폴더 안에 든 레시피에는 "이 프로그램을 자동 실행하라"는 내용이
들어갈 수 있습니다. 인터넷에서 받은 낯선 프로젝트가 나쁜 명령을 몰래
넣어놨을 수도 있겠죠. 그래서 **처음 여는 프로젝트의 레시피는 자동 실행하지
않고, "이 프로젝트를 신뢰하시겠습니까?" 창을 먼저 띄웁니다.**
낯선 USB를 컴퓨터에 꽂기 전에 한 번 확인하는 것과 같은 절차입니다.
한 번 신뢰하면 다음부터는 묻지 않습니다.

## 3. 새 기능 ③: AI 비서들끼리 협업시키기 (자동 프롬프트 전달)

### 그림으로 먼저

```
[칸 1: Claude Code가 코드 작성 중]        [칸 2: Codex 대기 중]
              │                                   ▲
              ▼                                   │
       파일이 바뀜! ──▶ [감시원이 발견] ──▶ "변경분 리뷰해줘" 메시지 전달
                              │
                        (전달 전에 사용자에게
                         [승인] / [무시] 버튼)
```

### 무엇이 되나

터미널 안에서 AI 도구(Claude Code, Codex 등)를 여러 개 동시에 띄워놓고
쓰는 상황을 가정합니다. 이때 **"감시원(watcher)"** 을 붙일 수 있습니다.

- 감시원은 정해진 것을 지켜봅니다. 예: "이 프로젝트 폴더의 파일 변경"
- 변화가 생기면 **정해진 규칙**에 따라 행동합니다.
  예: "Codex가 떠 있는 칸에 '방금 바뀐 코드를 리뷰해줘'라고 입력해 줘"

즉, 사람이 두 AI 사이에서 말을 옮기던 일을 프로그램이 대신합니다.

### 안전장치 (이 기능의 핵심은 편리함보다 안전)

남이(자동화가) 내 명령창에 마음대로 타이핑하는 것은 위험할 수 있습니다.
그래서 다음 장치가 **전부 기본으로** 들어갑니다.

1. **허락한 칸에만**: 칸마다 "자동 입력 허용" 스위치가 있고, **기본은
   꺼짐**입니다. 켜놓은 칸에만 자동 입력이 갈 수 있습니다.
2. **끼어들기 금지**: 대상 칸이 한창 일하는 중(출력이 나오는 중)이면
   기다렸다가, 조용해진 뒤에만 입력합니다. 통화 중인 사람에게 말 걸지 않는
   것과 같습니다.
3. **승인 버튼이 기본**: 규칙이 발동해도 바로 입력하지 않고
   "Codex 칸에 리뷰 요청을 보낼까요? [승인] [무시]" 알림을 먼저 띄웁니다.
   완전 자동은 사용자가 규칙마다 직접 켜야 합니다.
4. **기록장(감사 로그)**: 언제, 어떤 규칙이, 어느 칸에, 뭐라고 입력했는지
   전부 기록되고 나중에 열람할 수 있습니다.
5. **무한 반복 방지**: 자동 입력이 만든 출력에 다시 반응해서 뱅뱅 도는
   상황을 막기 위해, 한 번 발동한 규칙은 잠시 쉽니다.
6. **비상 정지 스위치**: 화면 하단 버튼 하나로 모든 자동화를 즉시 일시정지.

### 단계적으로 나옵니다

1. **1단계**: 수동 버튼 — 팔레트에서 "이 칸에 프롬프트 보내기"를 사람이
   직접 실행. (안전장치 먼저 완성)
2. **2단계**: 파일 변경 감시원 + 규칙 — 위 시나리오가 설정만으로 동작.
3. **3단계**: 외부 연결 통로 — 프로그래밍을 아는 사용자가 직접 만든
   "감시원 프로그램"을 terminal-f에 연결할 수 있는 공식 통로(API)를 엽니다.
   terminal-f의 플러그인 생태계의 시작점입니다.
   (3단계도 1번의 안전장치를 그대로 통과해야 하며, 우회할 수 없습니다.)

## 4. 자주 나올 질문 (FAQ)

**Q. 다른 책상으로 옮기면 원래 책상의 AI 작업이 멈추나요?**
A. 아니요. 화면만 안 보일 뿐 계속 돌아갑니다(keep-alive). 돌아오면 그동안
나온 출력이 밀린 것까지 다시 보입니다. 너무 오래·많이 쌓이면 가장 오래된
부분은 잘려나가고, 잘렸다는 표시가 뜹니다.

**Q. 자동 입력이 무섭습니다. 안 쓰면 그만인가요?**
A. 네. 모든 스위치의 기본값이 "꺼짐"이라, 아무 설정도 안 하면 지금과 완전히
같게 동작합니다.

**Q. 매크로 레시피가 이상한 명령을 실행하면요?**
A. 프로젝트에 딸려온 레시피는 신뢰 확인 전에는 어떤 명령도 자동 실행하지
않습니다. 내가 직접 만든 레시피는 내가 넣은 명령만 실행합니다.

**Q. 프로그램을 끄면 돌아가던 것들은?**
A. 책상 배치는 저장되어 다음에 그대로 복원되지만, 돌아가던 프로그램 자체는
종료됩니다. 다시 켜면 각 칸에 새 명령창이 준비됩니다.

## 5. 스크린샷 붙여넣기와 파일 끌어다 놓기

- **스크린샷 붙여넣기**: `Win+Shift+S`로 화면을 캡처한 뒤, 칸을 클릭하고
  `Ctrl+V`를 누르면 이미지가 파일로 저장되고 **그 파일의 경로**가
  입력됩니다. Claude Code 같은 AI 도구는 이 경로를 받으면 이미지를 첨부한
  것으로 처리합니다. (일반 텍스트 복사·붙여넣기는 그대로 동작합니다.)
- **파일 끌어다 놓기**: 탐색기에서 파일을 원하는 칸 위로 끌어다 놓으면 그
  파일의 경로가 입력됩니다.

## 6. 명령 팔레트 메뉴 사전

`Ctrl+Shift+P`를 누르면 뜨는 검색창의 **모든 메뉴**를 설명합니다.
몇 글자만 쳐도 검색되고, 방향키+Enter로 실행합니다.

### 6.1 화면(칸) 다루기 — 매일 쓰는 것

| 메뉴 | 하는 일 |
|---|---|
| **Pane: Split left/right** (`Ctrl+Shift+D`) | 지금 보고 있는 칸을 좌/우 둘로 나눔 |
| **Pane: Split top/bottom** (`Ctrl+Shift+-`) | 지금 칸을 위/아래 둘로 나눔 |
| **Pane: Close focused** (`Ctrl+Shift+W`) | 지금 칸을 닫음 (마지막 하나는 안 닫힘) |
| **Pane: Toggle zoom** (`Ctrl+Shift+Z`) | 지금 칸을 전체화면으로 확대 ↔ 원래대로 |
| **Workspace: New** | 새 책상(워크스페이스) 만들기 |
| **Workspace: Switch to "이름"** | 그 이름의 책상으로 이동 (`Ctrl+1~8`도 됨) |
| **View: Toggle sidebar** (`Ctrl+Shift+B`) | 왼쪽 사이드바 접기/펴기 |
| **View: Increase/Decrease font size** | 글자 크게/작게 |
| **Theme: Dark, Light, …** | 색 테마 바꾸기 |

### 6.2 매크로(레시피) — ⏳ 안정화 후 본격 사용 예정

책상 세팅을 자동화하는 기능입니다(§2 참고). 지금은 잘 안 쓰더라도,
프로젝트가 정착되면 아침마다 손으로 하던 세팅을 이걸로 대체하게 됩니다.

| 메뉴 | 하는 일 |
|---|---|
| **Template: Apply "이름"** | 저장된 레시피 실행 → 칸 나누기+폴더 이동+프로그램 실행이 완성된 **새 책상**이 뜸. 레시피에 물음표(`${repo}` 등)가 있으면 폴더 등을 물어봄 |
| **Template: Save current layout as template** | 지금 책상 배치를 레시피로 저장. 이후 팔레트에 "Apply "이름""으로 나타남 |
| **Template: Apply repo profile (.terminal-f/profile.json)** | 프로젝트 폴더 안에 든 공유 레시피 실행. 자동 실행 명령이 들어 있으면 **신뢰 확인 창**이 먼저 뜸(§2의 안전장치) |

### 6.3 자동 입력(주입) — ⏳ 안정화 후 본격 사용 예정

자동화가 특정 칸에 대신 타이핑해 주는 기능의 **수동 조작/안전장치**입니다
(§3 참고). 아무것도 안 켜면 완전히 꺼져 있는 것과 같습니다.

| 메뉴 | 하는 일 |
|---|---|
| **Injection: Allow/disallow on focused pane** | 지금 칸의 "자동 입력 허용" 스위치 켜기/끄기. 켜면 칸 이름표에 ⚡ 표시. **기본은 꺼짐** |
| **Pane: Edit labels (focused)** | 지금 칸에 이름표(라벨) 붙이기 (예: `codex`). 자동화가 "codex 칸"처럼 이름표로 대상을 찾음 |
| **Injection: Send prompt to pane N [라벨]** | 사람이 직접 특정 칸에 프롬프트를 보내는 수동 버튼. 허용 스위치가 켜진 칸에만 동작 |
| **Injection: Pause all (kill switch)** | 비상 정지. 모든 자동 입력을 즉시 일시정지 (내가 직접 치는 키보드는 영향 없음) |
| **Injection: Show audit log** | 기록장 열람 — 언제, 누가(수동/규칙/외부), 어느 칸에, 뭐라고 입력했는지 전부 |

### 6.4 자동화 규칙(감시원) — ⏳ 안정화 후 본격 사용 예정

"무엇이 일어나면 → 어느 칸에 뭐라고 입력한다"는 약속을 등록합니다(§3 참고).
규칙이 발동해도 기본은 [승인]/[무시] 확인 창이 먼저 뜹니다.

| 메뉴 | 하는 일 |
|---|---|
| **Automation: Add git-review rule (watch folder → inject to label)** | "이 폴더의 코드가 바뀌면, 저 라벨 칸에 리뷰 요청을 보내라"는 규칙 만들기 |
| **Automation: Add timer rule (inject every N minutes → label)** | "N분마다 저 라벨 칸에 이 문구를 보내라"는 규칙 만들기 (정기 상태 점검 등) |
| **Automation: List rules** | 등록된 규칙 목록 보기 |
| **Automation: Run rule now — 이름** | 그 규칙을 기다리지 않고 지금 바로 발동시켜 보기 (테스트용) |
| **Automation: Enable/Disable rule — 이름** | 규칙 켜기/끄기 (지우지 않고 잠시 쉬게 하기) |
| **Automation: Remove rule — 이름** | 규칙 삭제 |

### 6.5 외부 연결(고급) — ⏳ 개발자용, 안정화 후 확장 예정

프로그래밍을 아는 사용자가 직접 만든 감시원 프로그램(브로커)을 terminal-f에
연결하는 통로입니다(§3의 3단계). 일반 사용은 몰라도 됩니다.

| 메뉴 | 하는 일 |
|---|---|
| **Observe: Allow/disallow output observation on focused pane** | 외부 프로그램이 이 칸의 **화면 출력을 읽어가도 되는지** 스위치. 켜면 칸 이름표에 👁 표시. 화면에는 비밀번호 등이 지나갈 수 있으므로 **기본은 꺼짐** |
| **Control API: Show connection info (named pipe)** | 외부 프로그램이 접속할 통로 주소와 안내 표시 (비밀 토큰 자체는 화면에 안 보여줌) |

## 7. 한 줄 용어 사전

- **워크스페이스**: 프로젝트별 작업 책상
- **페인(pane)**: 책상 위 화면 분할 칸
- **매크로 / 프로필**: 책상 세팅 자동화 레시피
- **감시원(watcher)**: 뭔가(파일 변경 등)를 지켜보다 알려주는 자동 장치
- **규칙(rule)**: "무엇이 일어나면 → 어느 칸에 무엇을 한다"는 약속
- **자동 입력(프롬프트 주입)**: 자동화가 특정 칸에 대신 타이핑해 주는 것
- **감사 로그(audit log)**: 자동 입력의 전체 기록장
- **신뢰 확인(trust)**: 낯선 프로젝트의 자동 실행을 막는 첫 관문
- **라벨(label)**: 칸에 붙이는 이름표. 자동화가 "codex 칸"처럼 이름표로
  대상을 찾음
